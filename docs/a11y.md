# Accessibility Baseline (WCAG 2.1 AA)

**Status:** Foundation draft (Batch 1a, task 5). Every later UI task references this.

Non-technical genealogists include older users, users with limited vision,
users on slow connections, users on touch devices. The whole product fails if
they can't use it. This doc defines the floor.

Target: **WCAG 2.1 Level AA**. Tested with axe-core in CI; smoke-tested with
NVDA (Windows) + VoiceOver (macOS); keyboard-only walked monthly.

---

## The eight non-negotiables

1. **Contrast 4.5:1** for body text, **3:1** for large text (≥18pt or 14pt
   bold) and meaningful UI graphics. Verified at design-token level: every
   color pair in `tokens.css` ships a tested ratio.
2. **Keyboard reachability.** Every interactive element reachable via Tab in
   visual order. No keyboard traps (modals/lightboxes return focus on close).
3. **Visible focus ring.** Never `outline: none` without an accessible
   replacement. Default ring: 2px solid `--gold` with 2px offset.
4. **Touch targets ≥ 44×44 px** (Apple HIG / WCAG 2.5.5). Smaller chips get
   padding to hit-area without changing visual size.
5. **Semantic HTML first, ARIA only as patch.** A `<button>` beats
   `<div role="button">`. ARIA labels added only when semantic HTML can't
   carry the meaning (e.g. icon-only buttons).
6. **Alt text on every meaningful image.** Person photos use
   `photo_assertions[].alt_text` from the schema. Decorative images get
   `alt=""`. Never let alt default to filename.
7. **Respect `prefers-reduced-motion`.** Skeleton shimmer, expander animation,
   pan-zoom inertia — all gate on the media query and reduce to instant.
8. **Respect `prefers-color-scheme`.** Dark mode is real, not an afterthought.

## Color & contrast checklist (token-level)

| Token pair | Ratio | Use |
|---|---|---|
| `--ink` on `--paper` | 14.2:1 | Body text. Pass AAA. |
| `--gold-dark` on `--paper` | 4.8:1 | Primary links. Pass AA. |
| `--muted` on `--paper` | 5.1:1 | Secondary text. Pass AA. |
| `--paper` on `--gold` | 4.6:1 | Gold CTA button text. Pass AA. |
| `--ink` on `--gold` | 11.3:1 | Alt CTA pattern. Pass AAA. |
| dark-mode pairs | verified at build | Same thresholds. |

`scripts/check_contrast.py` re-verifies on token changes (added in Batch 1c
task 9).

**Confidence color scale** specifically: the bands must be distinguishable
without relying on color alone. The chip always includes the **band-name
label** ("Strong," "Moderate," etc.) so a user who sees only grayscale still
reads the meaning. The label is the primary signal; color is the accelerant.

## Keyboard interaction map (the global contract)

| Key | Effect |
|---|---|
| `Tab` / `Shift+Tab` | Move focus forward/back through interactive elements |
| `Enter` | Activate the focused button/link |
| `Space` | Toggle checkbox, activate button, scroll page (when no input focused) |
| `Esc` | Close modal, lightbox, expander; cancel form |
| `Arrow keys` | Move within composite widgets (pedigree nodes, fan-chart arcs, photo lightbox) |
| `Home` / `End` | Jump to first / last in a list or chart |
| `/` | Focus the global search (genealogy convention; opt-out per page) |
| `g + d` | Go to dashboard (Vim-style; defer to v2) |

### Focus order rule

Visual order = DOM order = focus order. If we're tempted to use `tabindex` to
reorder, we redesign instead.

### Focus return

Closing a modal or lightbox returns focus to the element that opened it. If
that element no longer exists (e.g. row was deleted), focus moves to the
nearest sensible parent.

## Screen-reader checklist

### Page-level

- Every page has one `<h1>` matching the page title.
- Heading hierarchy is unbroken (no `<h1>` → `<h3>` skips).
- The main content has `role="main"` (or `<main>`).
- Skip-to-content link is the first focusable element on every page.

### Component-level

- Person cards announce: "Person link, William Bailey Maxwell, born 1821,
  died 1895, confidence near-certain."
- Confidence chips announce: "Confidence: Strong, 89%." (Band + percentage,
  not just the percentage.)
- Possibility expanders announce: "Father, 10 possibilities. Activate to see
  candidates." Expanded: "1 of 10, Joseph Maxwell, born 1775, confidence
  moderate, 0.42."
- Conflicting assertions announce both values + both confidences + "both kept;
  see sources."
- Photos announce their `alt_text`. If `alt_text` is empty, the photo is
  decorative and not announced.

### Live regions

- Save confirmations use `aria-live="polite"` and read: "Saved — 3 sources
  added to William Maxwell."
- Errors use `aria-live="assertive"` and read the full error pattern from
  `microcopy.md`.

## Mobile / touch

- Touch targets **≥ 44×44 CSS pixels** (matches both Apple HIG and WCAG 2.5.5).
- Pinch-to-zoom **never disabled** at the meta-viewport level. Users may need
  to zoom regardless of our responsive layout.
- Drag gestures on pedigree / fan charts have a single-finger equivalent
  (button or arrow keys via on-screen controls).
- No hover-only affordances. Every hover state has a focus equivalent and a
  tap equivalent.

---

## Cognitive accessibility

- Plain English everywhere (avg reading grade ~8 in body copy).
- One primary action per screen. Multiple options are fine; multiple
  *equally prominent* actions are not.
- Confirmations restate the action (per `microcopy.md`).
- Errors name the cause and the next step.
- Time-out warnings (when we add session expiry): announce 60 seconds before;
  let the user extend.
- No flashing content >3 Hz (WCAG 2.3.1).

## Motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

Goes in `tokens.css`. Specific components opt back in only for state that
*requires* motion (e.g. skeleton shimmer becomes instant, but the loading
sentinel still exists).

## Forms

- Every input has a visible `<label>` (placeholder text does not count).
- Required fields marked with both `required` attribute AND a visible label
  pattern ("Birth year *").
- Errors appear next to the field, not just at the top.
- `aria-describedby` ties hint text and error text to the input.
- Autocomplete attributes used aggressively (`autocomplete="given-name"`
  etc.) — saves typing for everyone, especially screen-reader users.

## Testing requirements (gates for shipping a component)

Before any `<maxgen-…>` component lands on `main`:

| Gate | How |
|---|---|
| axe-core: 0 violations | Automated in `/dev/components` storybook page |
| Keyboard walkthrough | Manual: Tab, arrows, Enter, Esc — verify all interactions |
| Screen-reader spot-check | NVDA on Windows, monthly cycle |
| Contrast | `check_contrast.py` passes for every token pair the component uses |
| Touch target audit | DevTools mobile emulator, hit areas ≥ 44 px |
| Reduced motion | Toggle `prefers-reduced-motion: reduce` in DevTools, verify no required motion |
| Dark mode | Toggle, verify all states render |

A component without all seven is `pending` in the tracker, not `complete`.

## What this doc does NOT yet cover (deferred)

- Internationalization (RTL languages, locale-aware date formats, plural
  rules) — Batch 4 task 40 stubs the structure; full RTL audit is a v2 task.
- Voice control / dictation accuracy — needs real user testing.
- Captioning for any video content — we have none yet; rule when we add: all
  video has captions, no exceptions.
- Cognitive assistance overlays (read-aloud, simplified mode) — v2.
