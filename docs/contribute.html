What Is a Quality Checker?

After a worker submits a batch of records, a second contributor reviews them for accuracy — verifying names, dates, confidence scores, and that no data was hallucinated. Checkers earn 10% of every move they verify. Checkers must maintain a passing rate of 90%+ or they are removed from the queue. High-performing checkers can earn a steady income reviewing other contributors' work without running GPU-intensive embeddings.

Paid compute is coming in Phase 2. Phase 1 (now) is volunteer embedding — building the database. Phase 2 launches paid research tasks when the user base reaches critical mass. Join the paid contributor waitlist now to be first in the queue.

What You Need

GPU (recommended)

Any NVIDIA GPU with 8 GB+ VRAM runs embeddings ~10× faster than CPU. RTX 3060 or better is ideal. CPU works fine for scraping-only collections.

Python 3.11+

The pipeline requires Python 3.11 or newer. Dependencies install via pip install -r requirements.txt. A virtual environment is strongly recommended.

~100 GB disk space

Intermediate files and embeddings accumulate during a run. Most collections need 20–80 GB of scratch space depending on size.

Stable internet

Scraping collections pull from public APIs and archives. Your connection only needs to be reliable — bandwidth requirements are modest.

How to Contribute
1
Read the Extraction Rules — required before starting. They define license compliance, quality thresholds, and what gets accepted.
2
Clone the repo and configure your .env. See RESUME.md for full setup. Takes about 10 minutes.
git clone https://github.com/opengenealogyai/opengenealogyai pip install -r requirements.txt
3
Pick a collection below and claim it. The checkout system locks it for 30 days — no one else can claim the same collection while you're working it.
python -m pipeline.checkout claim <id>
4
Run the pipeline. It handles fetching, OCR, embedding, and validation automatically. Come back when it's done.
python -m pipeline.run --collection <id>
5
Submit when done. Records go through automated validation — minimum 60% quality pass, maximum 5% hallucination rate. Accepted submissions are embedded immediately.
python -m pipeline.checkout submit <id>
You get full attribution. Every record you embed carries your name and contributor URL in the Qdrant vector database — permanently. When a user's research report cites that record, your name is in the citation chain. Your work is part of the public record.
What the Pipeline Does
# Step 1 — Fetch raw documents from the source (archive APIs, scraping) python -m pipeline.fetchers.<source> --collection <id> # Step 2 — Run OCR and transcription on image-based documents python -m pipeline.workers.ocr --collection <id> # Step 3 — Validate records against extraction rules (quality_guard) python scripts/validate_submission.py submission.jsonl --collection-id <id> # Step 4 — Embed accepted records into Qdrant vector DB python -m pipeline.workers.embed --collection <id> # All steps run automatically via: python -m pipeline.run --collection <id>

Quality Gates

Every submission passes through automated validation before it enters the database. There are two hard rules:

60% minimum pass rate

At least 60% of records in your submission must pass quality checks — correct schema, source URL present, extraction confidence within range, license valid.

5% maximum hallucination rate

The hallucination guard checks that extracted text actually appears in source documents. If more than 5% of records fail this check, the whole submission is rejected.

Accepted → database immediately

Passing submissions are embedded into the vector database automatically. No manual review step. You'll see the count increment within minutes of submitting.

Rejected → returned with report

Rejected submissions include a full report showing which records failed and why. Fix the issues and resubmit — there's no penalty for resubmission.
